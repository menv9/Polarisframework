## 8. Decisiones del Sistema

### 8.1 Triggers automáticos

```text
PARAR OPERATIVA inmediatamente si:
  - DD > 25% (Stop level Risk Mgmt)
  - 5 trades consecutivos perdedores con override discrecional
  - Sharpe rolling 6m < 0 durante 2 meses consecutivos
  - IC realizado < 0 durante 6 meses

REDUCIR SIZING 50%:
  - DD entre 10-15% (Reduce 1)
  - Sharpe rolling 6m < 0.3 durante 1 mes
  - Profit factor rolling 30 trades < 1.0
  - Atribución del bloque dominante negativa 2 trimestres

RECALIBRAR (parcial):
  - β de un indicador cambia de signo
  - Una correlación driver-divisa cae > 50% vs histórico
  - Un régimen "imposible" persiste > 6m (ej: Mixto > 80% del tiempo)

CONTINUAR FULL:
  - Todas las métricas dentro de rangos
  - Sin triggers de los anteriores
```

### 8.2 Override discrecional: política

```text
Override = ejecutar contrario a la señal del sistema o saltarse reglas.

POLÍTICA:
  - Permitido solo en eventos catastróficos NO modelados
    (guerra mundial, default soberano major, ruptura monetaria)
  - Documentar por escrito ANTES de ejecutar
  - Trackear separately P&L de overrides
  - Si overrides P&L < sistema P&L → eliminar overrides
  - Más de 3 overrides en 12 meses → revisar disciplina, posible pausa

NO PERMITIDO:
  - Override por "sensación"
  - Override post-pérdida grande para "recuperar"
  - Override por noticias intra-día
  - Saltarse stops "porque va a volver"
```

### 8.3 Triage diagnóstico: distinguir tres situaciones operativas

> **Crítico:** la decisión correcta ante un período de bajo rendimiento depende de **diagnosticar correctamente la causa**. Confundir "mala racha normal" con "sistema roto" lleva a recalibraciones innecesarias (overfit). Confundir "régimen adverso" con "mala racha" lleva a perder más capital antes de reducir riesgo.

#### 8.3.1 Las tres situaciones

```text
SITUACIÓN A — Mala racha estadística normal
  Naturaleza:     varianza esperada del sistema, dentro de parámetros
  Acción:         NO HACER NADA, mantener disciplina
  Error típico:   "esto no funciona, hay que cambiar algo" → overfit destructivo

SITUACIÓN B — Régimen de mercado adverso
  Naturaleza:     el sistema está bien, pero el régimen actual no le encaja
  Acción:         REDUCIR SIZING temporalmente, esperar normalización
  Error típico:   continuar con sizing pleno → drawdown profundo evitable

SITUACIÓN C — Sistema fundamentalmente roto
  Naturaleza:     edge perdido (β obsoletos, supuesto fallido, drift estructural)
  Acción:         PARAR operativa, recalibración / revisión profunda
  Error típico:   continuar esperando "que vuelva" → ruina
```

#### 8.3.2 Matriz diagnóstica

Aplicar las 6 preguntas tras un período de bajo rendimiento (DD > 10% o Sharpe rolling 6m < 0):

| Pregunta | Mala racha (A) | Régimen adverso (B) | Sistema roto (C) |
|---|---|---|---|
| ¿IC realizado 6m? | > 0.03 | 0.00 a 0.03 | < 0 |
| ¿β del modelo siguen estables? | Sí | Sí | NO (alguno cambió signo) |
| ¿Hit rate por bloque? | Distribuido normal | Concentrado en 1-2 bloques fallando | Todos los bloques fallan |
| ¿Régimen de mercado vs típico? | Normal | Atípico (vol extrema, correlación rara) | Cualquiera |
| ¿Performance por par? | Distribuido | Concentrado en pares específicos | Mal en TODOS los pares |
| ¿Atribución vs backtest? | Coherente | Diverge en 1-2 factores | Diverge en mayoría de factores |

**Regla de decisión:**
```text
Mayoría de respuestas "columna A" → SITUACIÓN A
Mayoría "columna B"               → SITUACIÓN B
Mayoría "columna C" o ≥ 2 columna C → SITUACIÓN C
```

#### 8.3.3 Indicadores cuantitativos auxiliares

```text
DRIFT del IC:
  IC_actual − IC_histórico
  > −0.02   → varianza normal (Situación A)
  −0.05 a −0.02 → posible régimen (Situación B)
  < −0.05   → posible ruptura (Situación C)

ESTABILIDAD de β (test §17.4):
  Si todos los β rolling 12m vs 36m difieren < 30% → estable (A o B)
  Si > 30% en mayoría → inestabilidad estructural (C)

DRAWDOWN vs backtest:
  DD actual ≤ peor DD histórico (backtest) → A o B
  DD actual entre 1-1.5× peor DD histórico → B probable
  DD actual > 1.5× peor DD histórico → C muy probable

DURACIÓN del underperformance:
  < 3 meses    → A (varianza normal)
  3-9 meses    → B probable
  > 9 meses    → C probable, o régimen muy persistente
```

#### 8.3.4 Acción específica por situación

```text
SITUACIÓN A — Mala racha normal:
  - Continuar operativa con sizing actual
  - Resistir la tentación de cambiar el sistema
  - Documentar la racha en post-mortem mensual
  - Verificar que disciplina y ejecución siguen impecables
  - Esperar regression to the mean

SITUACIÓN B — Régimen adverso:
  - Reducir sizing 50% (al nivel "Reduce 1" de Risk Mgmt §6.1)
  - Identificar QUÉ régimen está activo y POR QUÉ no encaja
  - Revisar si World View detecta el régimen correctamente
  - Mantener sistema sin cambios estructurales
  - Re-incrementar sizing cuando régimen normalice (no antes)
  - Documentar el régimen y la respuesta del sistema (datos para futuro)

SITUACIÓN C — Sistema roto:
  - PARAR operativa nueva inmediatamente
  - Mantener solo posiciones abiertas con trailing apretado
  - Revisión profunda obligatoria:
      ☐ Re-correr regresión §9 con datos actualizados
      ☐ Aplicar tests §17 (anti-espuria) a los β actuales
      ☐ Re-evaluar supuestos económicos §16.8
      ☐ Verificar fuentes de datos (¿alguna serie cambió metodología?)
      ☐ Comparar con modelo congelado §16.7
  - Solo reanudar operativa con set de β recalibrado y validado OOS
  - Si supuestos económicos fallaron → reescritura, no parche
```

#### 8.3.5 Frecuencia de aplicación del triage

```text
Aplicación obligatoria del triage diagnóstico cuando:
  - DD > 10% (nivel "Reduce 1")
  - Sharpe rolling 6m < 0.3
  - Hit rate rolling 30 trades < 40%
  - 5 trades consecutivos perdedores

Aplicación recomendada (preventiva):
  - En cada post-mortem trimestral aunque las métricas estén bien
  - Tras cualquier cambio de régimen detectado por World View

NO aplicar el triage:
  - Tras un solo trade perdedor (varianza pura)
  - Tras 1 mes flojo (sample size insuficiente)
  - Bajo presión emocional post-pérdida (esperar 24-48h)
```

#### 8.3.6 Tabla resumen de acción por nivel de DD

| DD actual | Diagnóstico previo requerido | Acción default |
|---|---|---|
| 0-5% | No requerido | Continuar operativa normal |
| 5-10% | Triage informal (revisar últimos 10 trades) | Continuar, monitorizar |
| 10-15% | **Triage formal §8.3.2 obligatorio** | A: continuar / B: reducir 50% / C: parar |
| 15-20% | Triage formal + revisión escrita | A: reducir 50% / B: reducir 75% / C: parar |
| 20-25% | Triage formal + decisión documentada | Pausa siempre, decidir si recuperar tramo |
| 25%+ | Triage formal + revisión externa si posible | Stop total, revisión profunda obligatoria |
