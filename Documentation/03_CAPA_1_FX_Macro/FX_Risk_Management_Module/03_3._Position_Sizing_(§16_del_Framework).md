## 3. Position Sizing (§16 del Framework)

### 3.1 Tres métodos combinados

**Método 1 — Risk per Trade (hard cap):**
```text
Risk_$ = Capital × risk_pct        (típico: 1% por trade, máx 2%)
Pos_size = Risk_$ / (Stop_distance_pips × pip_value)
```

**Método 2 — Kelly Fraccional:**
```text
Edge estadístico (del backtest):
  p = probabilidad de ganar
  b = ratio average_win / average_loss
  q = 1 − p

Kelly óptimo:
  f* = (p·b − q) / b

Aplicación práctica (fractional Kelly por incertidumbre):
  f_aplicado = 0.25 × f*       (¼ Kelly recomendado)

Pos_size = Capital × f_aplicado / precio_actual
```

**Método 3 — Volatility Targeting:**
```text
Vol_objetivo_anual = 10-15% típico para FX sistemático

Pos_size = (Capital × Vol_objetivo) / (Vol_realizada_par × notional_unit)
         = Capital × Vol_objetivo / σ_par
```

### 3.2 Combinación operativa

```text
Pos_final = MIN(Pos_RiskPerTrade, Pos_Kelly, Pos_VolTarget)

Razón: el método más restrictivo manda. Hard cap evita ruina; Kelly evita
sobre-apalancamiento por edge sobreestimado; Vol-target normaliza entre pares.
```

### 3.3 Ajuste por convicción de señal

Multiplicador sobre `Pos_final`:

| Convicción de Señal_FX | Criterio | Multiplicador |
|---|---|---|
| Alta | \|Señal\| > 1.5·σ_señal | × 1.0 |
| Media | 1.0·σ ≤ \|Señal\| ≤ 1.5·σ | × 0.6 |
| Baja | \|Señal\| < 1.0·σ | × 0 (no operar) |

### 3.4 Ajuste por régimen de volatilidad

Aplicar multiplicador de §2.2 sobre `Pos_final` ya ajustado por convicción.

### 3.5 Ejemplo numérico end-to-end

```text
Capital = 100,000 USD
Par = EUR/USD
Señal_FX(EUR/USD) = +1.8·σ → convicción alta → mult = 1.0
Vol régimen = Normal → mult = 1.0
Risk per trade = 1% = 1,000 USD
Stop distance = 80 pips (técnico)
Pip value EUR/USD (lote estándar 100k) = 10 USD/pip

Método 1 (Risk per trade):
  Pos_RPT = 1,000 / (80 × 10) = 1.25 lotes estándar = 125,000 EUR

Método 2 (Kelly, asumiendo backtest p=0.55, b=1.5):
  f* = (0.55·1.5 − 0.45) / 1.5 = 0.25
  f_aplicado = 0.25 × 0.25 = 0.0625
  Pos_Kelly = 100,000 × 0.0625 / 1.0850 ≈ 5,760 EUR (muy bajo, Kelly conservador)

Método 3 (Vol-target):
  Vol_objetivo = 12% anual
  σ_EURUSD = 8% anual
  Pos_VolTarget = 100,000 × 0.12 / 0.08 = 150,000 EUR

Resultado:
  Pos_final = MIN(125,000, 5,760, 150,000) = 5,760 EUR

→ Kelly limita drásticamente. Si confías más en el edge,
  subir a ½ Kelly: 11,520 EUR. Si menos, mantener ¼.
```

> **Observación:** Kelly suele ser la restricción binding cuando el edge es modesto. Esto es **correcto** — protege contra over-leverage si tu edge es menor del que crees.
