## 5. Order Types

### 5.1 Tabla de uso

| Tipo de orden | Cuándo | Riesgo |
|---|---|---|
| **Market** | Liquidez alta + necesitas ejecución inmediata | Slippage en eventos |
| **Limit** | Quieres precio mejor que actual | No se ejecuta si mercado no llega |
| **Stop (Stop-Market)** | Stops y break entries | Slippage en gaps (devastador en eventos) |
| **Stop-Limit** | Como Stop pero con cap a slippage | Puede no ejecutar si gap supera límite |
| **OCO (One-Cancels-Other)** | Stop + target simultáneos | Estándar para cualquier trade abierto |
| **GTC (Good-Til-Cancelled)** | Mantener orden vigente días | Olvidar cancelar |
| **GTD (Good-Til-Date)** | Vigente hasta fecha | Más seguro que GTC |

### 5.2 Reglas de uso

```text
Stops protectivos: STOP-MARKET siempre (priorizar ejecución sobre precio)
Targets:           LIMIT siempre (priorizar precio sobre ejecución)
Entries:           MARKET en sesión líquida, LIMIT si tiempo lo permite
OCO obligatorio:   stop y target enviados juntos al broker
```

### 5.3 Trampa del stop-limit en stops protectivos

```text
NUNCA usar stop-limit como stop protectivo.

Escenario: long EUR/USD @ 1.0850, stop-limit 1.0700/1.0680.
Evento: gap a 1.0650 sin tocar 1.0700-1.0680.
Resultado: stop-limit NO ejecuta. Posición sigue abierta perdiendo.

Stop-market habría ejecutado a 1.0650 (slippage 50 pips, pero cerrado).
La diferencia entre slippage controlado y no-cierre es la diferencia entre
trade malo y ruina.
```
