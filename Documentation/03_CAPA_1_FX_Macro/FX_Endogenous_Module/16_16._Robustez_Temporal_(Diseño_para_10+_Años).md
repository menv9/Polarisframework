## 16. Robustez Temporal (Diseño para 10+ Años)

Esta sección blinda el modelo contra el paso del tiempo. Sin estas capas, el modelo decae notablemente a partir del año 4-7. Con ellas, sigue siendo replicable y operable a 10+ años.

### 16.1 Capa A — Umbrales de régimen como percentiles, no niveles

Sustituir umbrales absolutos (que asumen distribución actual de volatilidad) por percentiles rolling auto-adaptativos.

```text
Reemplazar:
  Risk-ON  si VIX < 18  AND  HY OAS < 400 bps
  Risk-OFF si VIX > 25  OR   HY OAS > 600 bps

Con:
  Risk-ON  si VIX  < percentil 30 (rolling 5Y)
        AND HY OAS < percentil 30 (rolling 5Y)
        AND S&P > 200dma rising
  Risk-OFF si VIX  > percentil 70 (rolling 5Y)
        OR  HY OAS > percentil 70 (rolling 5Y)
        OR  S&P < 200dma falling
```

Auto-recalibra a cualquier régimen estructural de volatilidad (alto en 2008-2011, bajo en 2017-2019, etc.).

### 16.2 Capa B — Re-calibración programada de β

```text
Cadencia obligatoria:
  • Cada 12 meses → re-correr regresión §9 con datos actualizados
  • Cada 36 meses → revisar set completo de indicadores

Triggers de revisión anticipada:
  • Sharpe OOS < 0.3 durante 2 trimestres consecutivos
  • Hit rate < 50% durante 6 meses
  • Cualquier β cambia de signo respecto al de hace 12 meses
  • Drawdown > umbral del backtest histórico
```

Si tras revisión el modelo no recupera métricas mínimas (§9.4) → **detener uso en producción** hasta diagnóstico.

### 16.3 Capa C — Abstracción sobre fuentes de datos

No hardcodear códigos de series en el modelo. Mantener un fichero externo de mapeo que se actualiza independientemente:

```yaml
# data_sources.yaml (estructura recomendada)
CPI:
  USA:
    primary:   { provider: FRED,     code: CPIAUCSL,        units: "Index 1982-84=100" }
    fallback:  { provider: BLS,      code: CUUR0000SA0 }
    transform: "yoy_pct"
  EUR:
    primary:   { provider: Eurostat, code: prc_hicp_manr,    units: "% YoY" }
    fallback:  { provider: ECB_SDW,  code: ICP.M.U2.N.000000.4.ANR }
    transform: "level"
  JPN:
    primary:   { provider: e-Stat,   code: 0003427113 }
    fallback:  { provider: BoJ,      code: PR01'PRCG15_CCCSAA }
    transform: "yoy_pct"
```

Cuando una fuente muere o cambia código → se edita solo el `.yaml`, el modelo no se toca.

### 16.4 Capa D — Health check automático mensual

Antes de generar señal, validar las 24 series para ambos países:

| Check | Regla | Acción si falla |
|---|---|---|
| Cobertura | ≥ 80% de datos no nulos en ventana 10Y | Excluir indicador del cálculo |
| Frescura | Última publicación < 90 días | Excluir, marcar warning |
| Continuidad | No hay saltos > 5σ vs σ histórica | Inspeccionar manualmente, posible cambio metodológico |
| Stationarity check | Augmented Dickey-Fuller p < 0.10 (tras transformación) | Revisar transformación §15.6 |
| Range sanity | Valor dentro de min/max histórico × 2 | Excluir, posible error de ingesta |

```text
Si > 30% de indicadores fallan health check para una divisa
  → No emitir señal para esa divisa ese mes (flat forzado)
```

### 16.5 Capa E — Reclasificación dinámica de divisas

El estatus refugio/mixto/pro-cíclica no es fijo. JPY ha sido refugio 1995-2020, pero la beta vs riesgo se está deteriorando (déficit por cuenta corriente, fin del YCC). Detectar mutaciones automáticamente:

```text
Cada 6 meses, calcular:
  β_riesgo(divisa) = cov(retornos_divisa, retornos_S&P500) / var(S&P500)
                     ventana = 3 años rolling

Reclasificación:
  β < −0.3              →  Refugio
  −0.3 ≤ β ≤ +0.3       →  Mixto
  β > +0.3              →  Pro-cíclica

Si una divisa migra de categoría → actualizar §15.1 y §15.3
```

### 16.6 Capa F — Protocolo de actualización de indicadores

Cada 3 años, revisar formalmente la lista de 24 indicadores:

| Categoría de cambio | Acción |
|---|---|
| Indicador deja de tener volumen / liquidez | Sustituir por proxy o eliminar |
| Nueva fuente más predictiva aparece | Evaluar inclusión vía regresión §9 |
| Cambio metodológico en serie existente | Re-bautizar como nueva serie, no encadenar |
| Régimen económico cambia (ej: fin de QE global) | Revisar todos los signos §4 |

Candidatos potenciales a evaluar en futuras revisiones:
- **Stablecoin flows** y volumen on-chain como proxy de demanda dólar
- **CBDC adoption rates** (especialmente CNY e-CNY)
- **Climate risk premium** sobre bonos soberanos
- **Demografía** (ratio dependencia) como driver estructural de tipos
- **Geopolitical risk index** (Caldara-Iacoviello)

Indicadores en riesgo de obsolescencia:
- **Breakevens TIPS** si reducen emisión
- **CFTC reports** si cambia régimen regulatorio (han cambiado formato 3 veces)
- **Balance CB/GDP** si bancos centrales abandonan estructuralmente QE

### 16.7 Capa G — Benchmark de modelo congelado

Mantener una **versión inmutable del modelo en t=0** corriendo en paralelo a la versión recalibrada:

```text
Modelos en producción:
  • Modelo_vivo:     recalibrado anualmente (capa B)
  • Modelo_congelado: snapshot inicial, nunca tocado

Comparación trimestral:
  Si Sharpe(Modelo_vivo, OOS últimos 3Y) < Sharpe(Modelo_congelado, OOS últimos 3Y)
    → La recalibración está introduciendo overfitting
    → Revertir Modelo_vivo a último β estable
    → Investigar causa
```

Esto detecta cuando los ajustes posteriores empeoran el modelo en lugar de mejorarlo.

### 16.8 Capa H — Auditoría de supuestos económicos

El modelo se sustenta en creencias económicas que pueden dejar de ser ciertas. Documentar explícitamente y revisar cada 3-5 años:

| # | Supuesto | Evidencia | Test de invalidación |
|---|---|---|---|
| 1 | Diferencial de tipos reales 2Y mueve FX a 1-3m | Engel 2014, Lilley 2022 | IC < 0.02 durante 3 años |
| 2 | Cuenta corriente ancla FX a 3-12m | Gourinchas-Rey | Correlación CC vs ΔFX 12m < 0.1 durante 5 años |
| 3 | Refugios se aprecian en risk-off | 1995-2024 | β_riesgo de USD/JPY/CHF se vuelve > 0 sostenidamente |
| 4 | Bancos centrales son driver dominante de FX | Post-Bretton Woods | Cuando intervención fiscal/política supere tipos como narrativa |
| 5 | Z-score 10Y captura régimen estructural | Auto-evidente bajo estacionariedad | Series con 2+ rupturas estructurales en ventana |
| 6 | Posicionamiento CFTC es contrarian en extremos | Datos 2000-2020 | Hit rate del signo CFTC < 50% durante 3 años |
| 7 | REER mean-reverts a 5-10 años | Rogoff 1996 | REER desviaciones > 20% se mantienen sin corregir > 7 años |
| 8 | Breakevens reflejan inflación esperada sin distorsión | Mercado líquido TIPS | Liquidity premium > 50bps sostenido |

**Si un supuesto falla → el modelo necesita reescritura, no parche.** Esto es lo que distingue un modelo de 10 años de un modelo que sobrevive un ciclo.

### 16.9 Eventos que ningún modelo sobrevive sin reescritura completa

Para honestidad intelectual, listar los escenarios que **invalidarían** el modelo entero, no solo lo recalibrarían:

- **Cambio de régimen monetario global:** vuelta a patrón oro, dominancia CBDC con programmable money, hiperinflación generalizada en G10.
- **Disolución / creación de bloques monetarios:** ruptura del euro, dolarización forzada en EM, bloque BRICS+ con moneda común.
- **Conflicto sistémico:** guerra entre potencias mayores, sanciones que rompan flujos de capital globales (precedente: Rusia 2022 a escala mayor).
- **Cambio tecnológico disruptivo:** liquidación 24/7 on-chain, desaparición del FX OTC tradicional.
- **Pérdida de privilegio del USD:** dedolarización efectiva del comercio internacional.

En estos casos, los z-scores 10Y incluirán datos de un régimen que ya no existe → **el modelo debe reconstruirse desde cero**, no recalibrarse.

### 16.10 Calendario de mantenimiento recomendado

| Frecuencia | Tarea |
|---|---|
| **Diaria** | Health check D, descarga de datos, cálculo de señal |
| **Mensual** | Recálculo de régimen (capa A), revisión de outliers |
| **Semestral** | Reclasificación de divisas (capa E) |
| **Anual** | Recalibración de β (capa B), comparación vs modelo congelado (G) |
| **Trianual** | Revisión de indicadores (capa F), auditoría de supuestos (capa H) |
| **Quinquenal** | Evaluación de obsolescencia estructural; decisión continuar vs reescribir |

### 16.11 Veredicto de longevidad esperada

| Configuración | Vida útil esperada |
|---|---|
| Modelo sin capas 16 (solo §1-15) | 2-4 años, decadencia notable a partir del 4 |
| Modelo con capas A+B+D | 5-8 años con monitoreo activo |
| Modelo con todas las capas A-H + §16.10 | **10+ años**, sujeto a §16.9 |

Asumiendo: (a) régimen FX flotante se mantiene, (b) bancos centrales siguen siendo el driver dominante, (c) no hay eventos sistémicos del tipo §16.9.
