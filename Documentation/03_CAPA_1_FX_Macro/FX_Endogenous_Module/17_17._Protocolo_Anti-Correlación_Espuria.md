## 17. Protocolo Anti-Correlación Espuria

> **Crítico:** se aplica **antes** de incluir cualquier driver en el modelo y **antes** de calibrar los β con datos reales. Las correlaciones espurias son el riesgo #1 cuando se calibra una regresión multivariante en finanzas; sin este filtro, los β capturan ruido histórico y el sistema falla out-of-sample.

### 17.1 El problema

Dos variables pueden correlacionar históricamente sin relación causal real (correlación espuria). En finanzas es muy común porque series macro y de mercado comparten factores latentes (ciclo global, dólar, tipos US). Si los β se calibran sobre relaciones falsas, el modelo funciona en backtest pero falla en vivo.

### 17.2 Test 1 — p-value de cada β

```text
Tras correr la regresión §9, verificar el p-value de cada coeficiente:

  p < 0.05  → menos del 5% de probabilidad de ser casualidad → ACEPTAR
  p > 0.05  → posible correlación espuria → DESCARTAR el driver

Aplicar criterio Bonferroni si se prueban muchos drivers:
  p_ajustado = 0.05 / N_drivers
  Con 24 drivers: p < 0.0021 para significancia robusta.
```

### 17.3 Test 2 — Justificación económica formal (escrita)

Antes de incluir cualquier driver, documentar por escrito:

```text
Driver:           [nombre]
Mecanismo:        ¿cuál es la cadena causal exacta entre el driver y la divisa?
Literatura:       ¿hay paper o teoría que respalde la relación?
Anti-tesis:       ¿en qué condiciones la relación se rompería?
Plazo del efecto: ¿inmediato, semanas, meses?

Si no hay mecanismo claro y articulable → NO INCLUIR aunque correlacione.
```

### 17.4 Test 3 — Estabilidad temporal del β

Dividir la ventana 10Y en dos subperíodos de 5Y:

```text
Subperíodo A (años 1-5):  β_A
Subperíodo B (años 6-10): β_B

Criterio:
  |β_A − β_B| / |(β_A + β_B)/2|  < 0.40   → ESTABLE, aceptar
  Diferencia 40-70%                       → INESTABLE, usar con cautela
  Diferencia > 70% o cambio de signo      → DESCARTAR

Ejemplo (válido):
  β_A(US10Y → EUR/USD) = −0.29
  β_B(US10Y → EUR/USD) = −0.31
  Diferencia: 7% → ESTABLE
```

### 17.5 Test 4 — VIF (Variance Inflation Factor)

Detecta multicolinealidad entre drivers (cuando dos drivers son tan correlacionados entre sí que la regresión reparte el β de forma inestable).

```text
Para cada driver i:
  VIF_i = 1 / (1 − R²_i)
  donde R²_i = R² de regresar driver_i contra todos los demás drivers.

Interpretación:
  VIF < 5    → aceptable, sin problema
  VIF 5-10   → precaución, considerar simplificar
  VIF > 10   → multicolinealidad GRAVE → eliminar uno de los drivers correlados

Casos sospechosos en el modelo actual (verificar antes de calibrar):
  - US 10Y nominal + US real rate 2Y     (EUR exo, AUD endo)
  - Iron ore + China PMI                 (AUD exo)
  - DXY + USD bias                       (varios pares)
  - VIX + HY OAS                         (filtro régimen, ya combinados)
```

### 17.6 Test 5 — Out-of-Sample obligatorio

```text
Procedimiento:
  1. Calibrar β con datos de los primeros 7 años (in-sample, IS).
  2. Generar señales con esos β en los últimos 3 años (out-of-sample, OOS).
  3. Comparar:
       Sharpe IS vs Sharpe OOS
       IC IS vs IC OOS
       
Criterios:
  Sharpe OOS / Sharpe IS > 0.6  → relación robusta
  Sharpe OOS / Sharpe IS < 0.3  → overfitting probable, descartar driver
  Sharpe OOS < 0                → driver capta ruido histórico, ELIMINAR
```

### 17.7 Test 6 — Correlación realizada vs esperada (en producción)

Una vez en vivo, monitorizar continuamente:

```text
Cada 3 meses:
  ρ_realizada = correlación entre Δdriver y Δdivisa últimos 3 meses
  ρ_esperada  = correlación histórica 10Y

Si |ρ_realizada − ρ_esperada| > 0.15 durante 2 ventanas consecutivas:
  → ALERTA: la relación puede haber cambiado
  → recalibrar β fuera del ciclo anual
  → revisar si el mecanismo económico (Test 2) sigue vigente
```

### 17.8 Checklist completa antes de incluir cualquier driver

```text
☐ p-value < 0.05 en regresión histórica (preferible Bonferroni)
☐ Justificación económica documentada por escrito
☐ β estable: diferencia entre subperíodos 5Y < 40%
☐ VIF < 5 (no multicolineal con otros drivers)
☐ Funciona OOS: Sharpe OOS / Sharpe IS > 0.6
☐ Correlación realizada vs esperada dentro de banda

Si NO cumple TODOS los 6 → driver NO entra al modelo.
Si cumple 5 de 6 → marcar como "watch list", revisar trimestralmente.
```

### 17.9 Drivers actualmente sospechosos en el framework

Riesgo de espuriedad / multicolinealidad a verificar antes de calibrar:

| Driver / par | Riesgo | Razón | Acción |
|---|---|---|---|
| US 10Y nominal + US real rate 2Y (EUR exo) | Alto | Muy correlados → VIF alto probable | Mantener solo uno (preferir real rate) |
| Iron ore + China PMI (AUD exo) | Medio | Ambos miden demanda China | Verificar VIF, posible doble cómputo |
| DXY + USD bias estructural | Medio | DXY incluye EUR/USD mecánicamente | Calcular DXY ex-target o usar otro proxy |
| VIX + HY OAS (filtro régimen) | Bajo | Ya combinados explícitamente como AND/OR | OK |
| Brent + NOK/CAD exógeno | Bajo | Mecanismo muy directo y documentado | OK |

### 17.10 Aplicación al pipeline de producción

```text
ANTES DE CALIBRAR (única vez al construir modelo):
  Para cada driver candidato → Tests 1, 2, 3, 4, 5
  → Lista final de drivers que pasan los 5

DURANTE PRODUCCIÓN (continuo):
  Test 6 cada 3 meses
  Alertas automáticas si correlación deriva
  
EN RECALIBRACIÓN ANUAL:
  Re-aplicar Tests 1, 3, 4, 5 con datos actualizados
  Cualquier driver que falle → bajar a watch list o eliminar

EN AUDITORÍA TRIANUAL:
  Re-aplicar Test 2 (justificación económica)
  ¿Sigue vigente el mecanismo? Si no → reescribir esa parte del modelo
```
