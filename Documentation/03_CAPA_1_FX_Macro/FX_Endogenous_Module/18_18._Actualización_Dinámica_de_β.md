## 18. Actualización Dinámica de β

> Complementa el §16.2 (recalibración anual). Permite ajustar β con mayor frecuencia para horizontes cortos y detectar obsolescencia antes del ciclo anual.

### 18.1 Tres métodos según horizonte operativo

| Método | Frecuencia | Apropiado para |
|---|---|---|
| **Recalibración anual fija** (§16.2) | 1× año | Horizonte 3-12m, sistemas estables |
| **Ventana rodante** | Mensual | Horizonte 1-3m, equilibrio simplicidad/responsividad |
| **Kalman Filter** | Continua (con cada dato) | Horizonte 1-4 semanas, sistemas dinámicos |

### 18.2 Ventana rodante (Rolling Window)

```text
Cada mes (último día):
  β_t = regresión sobre últimos N meses
  N típico:  36-60 meses (3-5 años)

Ventajas:
  - Simple de implementar
  - Captura cambios estructurales lentos
  - Estable, baja varianza β-a-β

Desventajas:
  - Lento ante cambios bruscos de régimen
  - Requiere mínimo 36 meses de historia para arrancar

Diferencia vs anual fija:
  Anual fija    → 1 cambio brusco al año
  Rolling 60m   → cambios graduales mensuales, mismo σ medio
```

### 18.3 Kalman Filter

```text
Para cada nueva observación (mensual o semanal):
  β_nuevo = β_anterior + K · (observado − predicho)
  
  donde K (Kalman gain) se ajusta automáticamente:
    Mayor K = más peso al dato nuevo (β reactivo)
    Menor K = más peso al β anterior (β estable)

Ventajas:
  - Adapta β en tiempo real a cambios de régimen
  - K se calibra automáticamente con la varianza observada
  - Apropiado para horizonte semanal donde correlaciones cambian rápido

Desventajas:
  - Más complejo de implementar (state-space model)
  - Riesgo de sobrerreacción si parámetros mal calibrados
  - Requiere validación rigurosa OOS

Recomendado solo si:
  - Horizonte operativo dominante < 4 semanas
  - Capa 2 (trend following corto) activa
  - Equipo técnico con experiencia en filtros de Kalman
```

### 18.4 Alerta automática de β obsoleto (independiente del método)

```text
Trigger de recalibración fuera de ciclo:

Calcular cada mes:
  ρ_3m  = correlación realizada (Δdriver, Δdivisa) últimos 3 meses
  ρ_10Y = correlación esperada (10Y rolling)
  
  drift = |ρ_3m − ρ_10Y|

Reglas:
  drift < 0.10            → normal, sin acción
  drift 0.10-0.15         → watch, monitorizar próximo mes
  drift > 0.15 durante 2 meses consecutivos
                          → ALERTA
                          → recalibrar β de ese driver inmediatamente
                          → no esperar al ciclo anual

Si el driver vuelve a alinearse tras 1-2 meses → falsa alarma, revertir.
Si la divergencia persiste > 6 meses → revisar mecanismo económico (Test 2 §17.3).
```

### 18.5 Recomendación operativa por capa del sistema

```text
Capa 1 — Macro 1-3 meses (sistema base):
  → Rolling Window 60 meses (mensual)
  → + alerta automática §18.4

Capa 2 — Trend following 1-2 semanas (futuro):
  → Kalman Filter (semanal)
  → + alerta §18.4 con ventanas más cortas (1m vs 3m)

Capa 3 — Equities macro (futuro):
  → Rolling Window 36 meses
  → + alerta §18.4

NUNCA mezclar métodos para el mismo driver.
Documentar el método aplicado por driver en data_sources.yaml (§16.3).
```

### 18.6 Backtest comparativo de métodos

Antes de decidir qué método aplicar, comparar sobre histórico:

```text
1. Reservar últimos 3 años como OOS estricto.
2. Calibrar con cada método sobre los 7 años previos.
3. Generar señales OOS con cada método.
4. Comparar: Sharpe, MDD, hit rate, IC, estabilidad de β.
5. Elegir el método con mejor combinación, no solo Sharpe máximo.

Trampa común:
  Kalman Filter mostrará Sharpe IS más alto SIEMPRE (sobreajusta).
  Lo que importa es Sharpe OOS y estabilidad fuera de muestra.
```
