## 11. Triggers críticos del sistema (resumen)

| Evento | Acción | Módulo |
|---|---|---|
| Convicción < 1σ | No abrir trade | Endo §15.7 |
| z_LF (CFTC) > +2 en dirección señal | Esperar purga | Timing §2 |
| Multi-timeframe contradice señal | Wait | Timing §6 |
| Evento NIVEL 1 en próximas 48h | Defer entry | Execution §8 |
| Spread > 1.5× típico | No ejecutar | Execution §11 |
| Edge_neto negativo (costes > edge) | Abort | Execution §1.4 |
| DD > 10% | Reduce 1 + triage formal | Risk §6 + SA §8.3 |
| DD > 20% | Pause + revisión obligatoria | Risk §6 |
| DD > 25% | Stop + auditoría | Risk §6 |
| Sharpe rolling 6m < 0 (2 meses) | Triage; posible Pause | SA §8.1 + §8.3 |
| IC < 0 durante 6 meses | Triage; posible sistema roto | SA §8.3 |
| β cambia de signo | Recalibración fuera de ciclo | Endo §16.2 + §18.4 |
| Drift correlación > 0.15 (2 meses) | Recalibrar β del driver | Endo §17.7 + §18.4 |
| Régimen extremo contra trade | Veto World View | WV §7.4 |
| Capa 1 flips estando Capa 2 abierta | Cerrar Capa 2 al open siguiente | Trend §1.3 |
| Capa 1 ya tiene par abierto y Capa 2 quiere mismo par | Sizing Capa 2 × 0.5 | Trend §4.2 |
| DD Capa 2 > 12% | Pause solo de Capa 2 | Trend §9 |
| DD Capa 3 > 20% | Pause solo de Capa 3 | Equities §9 |
| DD agregado (1+2+3) > 25% | Pause GLOBAL del sistema | Equities §9 |
| Earnings single stock < 2 semanas | No abrir Capa 3 single stock | Equities §2.3 |
| Health check fail rate > 30% | No emitir señal ese día | Implementation §3.3 |
| Drift correlación Kalman > 0.20 (3 sem) Capa 2 | Recalibración manual | Trend §6 |
| Operar EM (Tier 1-2) | Sizing 0.5%, convicción 1.8σ, w_exo 0.55-0.80 | EM §5 / §14 |
| EM TIER 3 (pegged) | NO operar direccional, solo opciones peg-break | EM §10.1 |
| EM TIER 4 (hyperinflation) | NO incluir en framework v1 | EM §10.2 |
| CDS EM > P90 5Y | NO abrir longs en esa divisa | EM §13 |
| Saldo broker extranjero > 50k EUR | Modelo 720 obligatorio (España) | Tax §4 |
| P&L anual > 200k sostenido | Evaluar SL operativa con asesor | Tax §5 |
| Cierre con pérdida en diciembre | NO recomprar mismo activo en 2 meses | Tax §3.2 |
| Pre-live deployment | Validar sistema contra Scenario Library | Scenarios §15 |
| Ocurre evento crítico nuevo | Documentar en Scenario Library < 30 días | Scenarios §16 |
| Falla broker / internet / hardware | Procedimiento failover §4 | Disaster Recovery §4 |
| Detectas señales rojas revenge trading | PARAR + cooldown 24h obligatorio | Behavioral §6.2 |
| 3+ overrides discrecionales en 1 mes | Pausa voluntaria del sistema | Behavioral §9 |
| > 12 meses sin pausa del sistema | Vacaciones forzadas 2 semanas | Behavioral §7 |
| Operar BTC/ETH | Sizing 0.25-0.5% RPT, k=4-5 ATR | Cross-Asset §9 |
| Operar commodities con futuros | Verificar curva contango/back antes | Cross-Asset §4.1 |
| Capital > 100k EUR | Considerar añadir broker secundario | Multi-Broker §1.3 |
| Margin usage > 80% en algún broker | Reducir exposición o transferir | Multi-Broker §9.2 |
| Saldo broker extranjero > 50k EUR | Modelo 720 (cuenta agregada multi-broker) | Multi-Broker §7.2 / Tax §4 |
