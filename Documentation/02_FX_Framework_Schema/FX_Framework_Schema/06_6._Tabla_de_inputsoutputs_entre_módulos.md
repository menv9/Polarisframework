## 6. Tabla de inputs/outputs entre módulos

| Módulo | Inputs | Outputs |
|---|---|---|
| **World View** | VIX, HY OAS, S&P trend, EMBI, GDP forecasts, encuestas, DXY, breakevens | régimen risk-on/off, USD bias, momentum global, régimen inflación |
| **Endogenous** | 24 series macro × país, ventana 10Y | Score_Endo(país) por horizonte |
| **Exogenous** | drivers externos por divisa (commodities, tipos US, etc.) | Score_Exo(país) |
| **Currency Views (combinación)** | Score_Endo + Score_Exo + WorldView state | Señal_FX(A/B) con convicción |
| **Timing** | Señal_FX, CFTC, técnico, price action, calendario | trade-now / wait / abort |
| **Risk Mgmt** | trade ejecutable + ATR + IV + capital + DD actual | size, stop, target, plazo |
| **Execution** | trade ejecutable | orden enviada + costes contabilizados |
| **Self-Awareness** | TODO lo anterior + P&L realizado | métricas, atribución, decisiones (continuar/reducir/parar/recalibrar) |
| **Trend Layer (Capa 2)** | Señal_FX Capa 1 + datos técnicos + ATR | trades 1-2 semanas con sizing reducido (0.5%) |
| **Equities Macro (Capa 3)** | Score macro Capa 1 + drivers exo equities | trades en ETFs/stocks 1-3 meses (0.75%) |
| **Implementation Stack** | Toda la lógica de los módulos | código Python, infra, automatización, dashboard |
| **Emerging Markets** | Datos EM-específicos (CDS, EMBI, reservas) | Score_FX para EM con sizing reducido y filtros adicionales |
| **Tax & Fiscal** | P&L bruto realizado, statements brokers | P&L neto after-tax, decisión vehículo, calendario fiscal |
| **Scenario Library** | Eventos históricos + sistema actual | Validación cualitativa pre-live, material de entrenamiento |
| **Disaster Recovery** | Eventos infra (corte, hack, broker down) | Procedimientos, contactos, kill switch |
| **Behavioral Finance** | Estado emocional, sesgos, journaling | Reglas anti-discreción, cooldowns, vacaciones forzadas |
| **Cross-Asset** | Drivers macro (real rates, DXY, VIX) | Score_Commodity / Score_Crypto, sizing reducido |
| **Multi-Broker** | Posiciones de cada broker individual | Master sheet agregado, exposición unificada |

**Flujo del feedback loop:**
```
Self-Awareness detecta degradación
    ↓
Triage A/B/C diagnóstico
    ↓
A → no acción (varianza normal)
B → reducir sizing (régimen adverso)
C → recalibrar / parar (sistema roto)
    ↓
Ajustes a módulos correspondientes:
  - β obsoleto         → Endogenous §18 recalibración
  - Régimen mal detectado → World View §3 ajuste umbrales
  - Setup técnico fallando → Timing §7 revisar criterios
  - Slippage alto       → Execution §6 cambiar broker
  - DD profundo         → Risk §6 nivel "Reduce" / "Pause"
```
