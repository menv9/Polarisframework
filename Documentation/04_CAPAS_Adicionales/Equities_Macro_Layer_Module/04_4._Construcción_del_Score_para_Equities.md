## 4. Construcción del Score para Equities

### 4.1 Score_Equity por instrumento

```text
Para cada ETF/stock:
  Score_Equity(X) = Σ ( z_driver_i · signo_i · β_i_para_X )

donde:
  z_driver_i = z-score del driver i (ya calculado para FX)
  signo_i    = signo del impacto del driver sobre X (long o short)
  β_i_para_X = sensibilidad histórica de X al driver (regresión propia)

Ejemplo BHP:
  β(iron_ore)       = +0.50
  β(China PMI)      = +0.30
  β(USD DXY)        = −0.20
  β(VIX)            = −0.25
  β(US 10Y real)    = −0.15
  
  Score_BHP = z(iron_ore)·0.50 + z(China_PMI)·0.30 − z(DXY)·0.20 − z(VIX)·0.25 − z(US10Y_real)·0.15
```

### 4.2 Decisión de trade

```text
Score_Equity(X) > +1.5σ_score → long X (convicción alta)
Score_Equity(X) > +1.0σ_score → long X (convicción media, sizing reducido)
|Score| < 1.0σ_score          → no operar X
Score_Equity(X) < −1.0σ_score → short X
Score_Equity(X) < −1.5σ_score → short X (convicción alta)

CRÍTICO: el score de equities se calibra con la misma metodología
que FX (regresión OLS, ventana 10Y, validación OOS, anti-espuria §17 endo).
```

### 4.3 Compatibilidad con Capa 1 FX

```text
REGLA: Capa 3 NO debería contradecir la dirección general del régimen.

Si Capa 1 FX dice "USD bullish + risk-off":
  Capa 3 debería estar long defensivos / short cíclicos.
  Si Score_Equity dice long XLB → conflicto interno → NO ejecutar.

Verificación pre-entry:
  ¿La posición Capa 3 es coherente con World View?
  ¿La posición Capa 3 es coherente con el régimen identificado?
  Si NO → revisar inputs antes de operar (probable error de cálculo o driver mal calibrado).
```
