## 1. Para qué sirve esta biblioteca

### 1.1 Tres usos primarios

```text
1. VALIDACIÓN cualitativa del sistema
   ¿Habría detectado el régimen risk-off?
   ¿Habrían disparado los circuit breakers a tiempo?
   ¿Cómo habrían reaccionado los β a la disrupción?

2. ENTRENAMIENTO del operador
   Cuando ocurra el próximo evento similar, tendrás contexto.
   Patrones repetibles facilitan respuesta correcta sin pánico.

3. STRESS TEST de implementación
   Si el sistema se implementa, verificar contra estos escenarios:
     - ¿Se identificó el régimen?
     - ¿Se aplicó el filtro World View?
     - ¿Se cerraron posiciones cuando debían?
```

### 1.2 Estructura de cada escenario

Cada escenario documentado con:
1. Fecha y duración
2. Contexto pre-evento
3. Disparador / catalizador
4. Movimientos de mercado
5. Comportamiento de drivers macro
6. Qué habría hecho el sistema (predicción cualitativa)
7. Circuit breakers / triggers que habrían activado
8. Lecciones específicas para el framework
