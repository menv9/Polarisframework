## 4. Procedimientos por escenario

### 4.1 Internet doméstico cae

```text
1. Activar hotspot móvil (4G/5G)
2. Verificar acceso a app broker
3. Si trade abierto: monitorizar, no operar nuevo
4. Si stop/target hit: confirmar ejecución desde app
5. Una vez restaurado internet principal: reconciliar posiciones
```

### 4.2 Broker plataforma down

```text
1. Verificar si es problema general (status page del broker, Twitter)
2. Llamar al dealing desk para gestionar manualmente si urgente
3. NO abrir nuevo trade hasta plataforma estable
4. Si stop crítico cerca: cerrar manual via teléfono
5. Documentar el incidente en journal
```

### 4.3 Apagón eléctrico prolongado

```text
1. UPS arranca (~30-60 min disponibles)
2. Decidir: ¿cerrar posiciones o esperar?
   - Si > 1h esperado: cerrar posiciones de mayor riesgo
   - Si < 30min: monitorizar
3. Móvil con datos como backup
4. Si UPS se agota: cierre de emergencia desde móvil
5. Apagado ordenado del sistema
```

### 4.4 Pérdida de hardware (ordenador / servidor)

```text
1. Acceso al broker via móvil/web → verificar posiciones
2. Cerrar manualmente las que requieran intervención
3. Suspender operativa nueva hasta restaurar hardware
4. Restaurar desde backups:
   - Sistema: clonar desde git
   - Datos: cargar desde cloud backup
   - Configuración: cargar desde git
5. Verificar integridad antes de reanudar trading automático
6. Tiempo típico de restauración: 24-72 horas
```

### 4.5 Hackeo / phishing broker

```text
ACCIÓN INMEDIATA (en este orden):
  1. Llamar dealing desk del broker → bloquear cuenta
  2. Cambiar password broker desde dispositivo limpio
  3. Cambiar password email asociado
  4. Revocar 2FA y reconfigurar
  5. Verificar todas las posiciones y movimientos recientes
  6. Si transferencias no autorizadas: denuncia policial inmediata
  7. Contactar asesor fiscal y abogado si transferencia material

PREVENCIÓN:
  ✗ Nunca click en enlaces de email "del broker"
  ✓ Acceder siempre via bookmark o tipeando URL
  ✓ 2FA activo siempre
  ✓ Antivirus actualizado
  ✓ NO usar redes WiFi públicas para trading
```

### 4.6 Broker quiebra

```text
PRECEDENTES: FXCM 2015, MF Global 2011, Lehman 2008, Alpari UK 2015

PRIMEROS PASOS:
  1. Verificar noticia oficial (no rumor)
  2. NO mover posiciones (puede empeorar situación legal)
  3. Documentar todas las posiciones y saldos al momento
  4. Identificar el "trustee" / administrador concursal
  5. Registrar como acreedor formalmente
  
RECUPERACIÓN:
  - Fondos segregados (FCA, NFA, CySEC) generalmente recuperables
  - Tiempo: 6 meses - 3 años típicamente
  - Recovery rate histórico: 80-100% en brokers regulados tier-1
  
PREVENCIÓN:
  ✓ Solo brokers regulados tier-1 (FCA, NFA, CFTC, ASIC)
  ✓ Verificar segregación de fondos
  ✓ Si capital > 100k: diversificar entre 2-3 brokers
  ✓ Monitorizar salud financiera del broker (rumors de problemas)
```

### 4.7 Operador enferma / incapacitado

```text
PROTOCOLO si > 1 semana fuera de servicio:

OPCIÓN A — Sistema completamente automatizado:
  - Sistema sigue solo
  - Alertas críticas configuradas a SMS/Telegram
  - Persona de confianza notifica si problema grave
  - Capacidad de cerrar todo via "kill switch" remoto

OPCIÓN B — Operativa requiere supervisión humana:
  - PARAR el sistema (cerrar todas las posiciones abiertas)
  - Mantener solo cash hasta recuperación
  - Coste de oportunidad: aceptable vs riesgo de fallo no atendido

ANTES DE QUE OCURRA:
  ☐ Documentar dónde están las cuentas, broker, asesor fiscal
  ☐ Persona de confianza tiene la información de contacto
  ☐ Procedimiento simple de "pausa total" documentado
  ☐ Acceso de emergencia delegado (con asesoría legal: poder notarial?)
```
