## 8. Lo que se Subestima Frecuentemente

### 8.1 Tiempo real de implementación

```
Estimación honesta:
  Código del MVP: 4-6 meses (vs los "6-8 semanas" que muchos dicen)
  Sistema completo: 8-12 meses de código
  Validación en vivo: 12-18 meses adicionales
  Total hasta sistema completamente validado: 2-3 años

La parte que tarda más es:
  - Aprender APIs y limpiar datos macro (revisiones, gaps, cambios metodológicos)
  - Walk-forward sin look-ahead (técnicamente delicado)
  - Debugging issues que solo aparecen con datos reales
  - Disciplina de no operar hasta tener calibración real
```

### 8.2 Disciplina psicológica

```
El framework asume que escribiendo reglas se siguen.
Realidad estadística:
  ~80% de operadores rompen sus propias reglas eventualmente
  Mayor probabilidad durante drawdown grande
  Mayor probabilidad después de racha buena (overconfidence)

Behavioral Module §8 ayuda pero NO garantiza.
Disciplina es habilidad construida con años, no rasgo poseído.
```

### 8.3 Costes reales operativos

```
Costes explícitos modelados (spread, swap, slippage, comisión).
Costes implícitos NO modelados:
  - Tiempo del operador (4-10h/semana)
  - Coste de oportunidad de no hacer otra cosa con esas horas
  - Estrés mental
  - Coste de infraestructura adicional (asesor fiscal especializado, etc.)
  - Tax-drag (no compensaciones perfectas entre años)
```

### 8.4 Sharpe esperado realista

```
Sistema retail individual con datos gratis:
  Sharpe esperado realista: 0.4-0.7 si bien calibrado
  Sharpe excepcional: 0.8-1.0
  Sharpe > 1.0 sostenido OOS: muy raro, sospechar overfitting

Hedge funds profesionales con Bloomberg + Reuters + datos institucionales:
  Sharpe sostenido en FX systematic: 0.5-0.9 típico
  
NO esperar Sharpe 1.5+ del framework. Si lo ves: AUDITAR antes de creer.
```
