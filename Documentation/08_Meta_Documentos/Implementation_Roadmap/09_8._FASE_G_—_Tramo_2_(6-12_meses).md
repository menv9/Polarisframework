## 8. FASE G — Tramo 2 (6-12 meses)

### Objetivo
Escalar a 50% del capital con sizing 0.75%. Validar escalabilidad del sistema.

### Setup financiero

```
CAPITAL ACTIVO:              50% del total = 25,000 EUR (de 50k ejemplo)
RIESGO POR TRADE:            0.75% × 25,000 = 187.50 EUR

CAMBIOS vs TRAMO 1:
  - Sizing 1.5× mayor por trade
  - Capital 2.5× más expuesto
  - Pero edge debería ser el mismo si nada cambió
```

### Vigilancia específica Tramo 2

```
CUIDADO: con sizing mayor, los errores cuestan más.
  - Slippage absoluto mayor (mismo % de notional pero notional mayor)
  - DD absoluto mayor (puede causar más estrés psicológico)
  - Stop temporal hits cuestan más en EUR

VERIFICAR:
  - ¿La performance live mantiene Sharpe del Tramo 1?
  - ¿El sizing mayor causa overrides emocionales?
  - ¿Hay efectos de market impact (improbable a este tamaño)?
```

### Decision gate G → H (DECISION GATE 3)
```
PARA AVANZAR A TRAMO 3 (operativa plena):
  ✓ Tramo 2 completado mínimo 6 meses (idealmente 12)
  ✓ Sharpe live mantiene > 0.4 con sizing mayor
  ✓ MDD < 18%
  ✓ Métricas no degradan vs Tramo 1
  ✓ Disciplina mantenida

  Si métricas degradan al escalar sizing:
    → Posible problema de psicología (no sistema)
    → Mantener Tramo 2 más tiempo
    → Trabajar disciplina antes de escalar más
```
