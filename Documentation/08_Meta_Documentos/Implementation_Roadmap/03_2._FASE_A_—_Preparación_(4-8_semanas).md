## 2. FASE A — Preparación (4-8 semanas)

### Objetivo
Setup infraestructural antes de escribir código. Sin esto, las fases siguientes se atascan.

### Deliverables

**Semana 1-2: Setup técnico**
```
☐ Python 3.11+ instalado con poetry
☐ Editor configurado (VSCode con Python extensions)
☐ Git instalado, cuenta GitHub privada
☐ Repo inicializado: fx_framework/ (estructura de Implementation Stack §2)
☐ Cuentas APIs:
   - FRED API key (gratis, registro inmediato)
   - Cualquier broker tier-1 demo account (IB recomendado)
☐ Editor de spreadsheet para tracking inicial (Excel/Google Sheets)
```

**Semana 3-4: Infraestructura física** - NO
```
☐ UPS para router + ordenador (~120 EUR)
☐ Móvil 4G/5G con plan de datos verificado
☐ Power bank cargado
☐ Gestor de contraseñas (Bitwarden gratis o 1Password ~30 EUR/año)
☐ Backup cloud (Google Drive/Dropbox) configurado
☐ Hardware mínimo: ordenador con 8GB RAM, SSD 256GB+
```

**Semana 5-6: Documentación pre-código**
```
☐ Leer los 18 módulos del framework completos
☐ Leer System_Weaknesses.md
☐ Definir EN PAPEL las fórmulas exactas que producirá el MVP:
   - ¿Qué 5-6 drivers para empezar?
   - ¿Qué peso inicial cada uno?
   - ¿Cómo se normaliza?
   - ¿Qué define "convicción alta/media/baja"?
☐ data_sources.yaml inicial con las series necesarias
```

**Semana 7-8: Asesoramiento profesional** -NO
```
☐ Contactar asesor fiscal especializado:
   - Reunión inicial (500-1000 EUR)
   - Definir estructura: persona física inicialmente
   - Confirmar Modelo 720 si aplica
☐ Verificar broker(s) elegido(s):
   - Regulación tier-1
   - Productos necesarios (FX + futuros + opciones si aplica)
   - Comisiones razonables
☐ Plan de capital total a comprometer al sistema
```

### Decision gate A → B
```
PARA AVANZAR A FASE B:
  ✓ Setup técnico funcional
  ~~✓ Infra física lista (UPS, móvil backup, etc.)~~
  ✓ Fórmulas del MVP en papel
  ~~✓ Asesor fiscal con quien consultar~~
  ~~✓ Capital reservado y aceptación de plan multi-año~~
  ✓ Compromiso real de 10-15h/semana (mínimo año 1)

SI ALGO FALTA: completar antes de seguir. NO acortar Fase A.
```
