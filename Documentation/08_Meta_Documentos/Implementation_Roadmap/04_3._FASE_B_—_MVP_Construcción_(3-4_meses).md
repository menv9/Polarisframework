## 3. FASE B — MVP Construcción (3-4 meses)

### Objetivo
Dashboard semi-automático de decisión (NO ejecución). Genera señales para que tú decidas operar manualmente.

### Mes 1: Pipeline de datos

```
SEMANA 1-2:
  ☐ Fetcher FRED funcional (10-15 series core)
  ☐ Fetcher yfinance funcional (precios FX, ETFs)
  ☐ Almacenamiento en Parquet local
  ☐ Tests básicos pytest

SEMANA 3-4:
  ☐ Health checks (cobertura, frescura, outliers)
  ☐ Transformaciones básicas (z-score 10Y rolling, YoY %)
  ☐ Fetcher ECB SDW (los datos europeos son los más difíciles)
  ☐ Cache de datos para no consumir API limits

DELIVERABLE FIN DE MES 1:
  Script daily_update.py que descarga las series
  y las guarda limpias localmente.
```

### Mes 2: Modelo mínimo

```
SEMANA 5-6:
  ☐ Cálculo de z-scores rolling 10Y
  ☐ Score_Endo para 2 países (USA, Eurozone) con 5-6 drivers
  ☐ Señal_FX EUR/USD básica
  ☐ World View básico (régimen risk-on/off con VIX + HY OAS)

SEMANA 7-8:
  ☐ Ampliar a 4 países (USA, EUR, JPY, GBP)
  ☐ Señal_FX para 6 pares G10
  ☐ Score con β placeholder (literatura)
  ☐ Logs estructurados (loguru)

DELIVERABLE FIN DE MES 2:
  Sistema que genera señales para 6 pares G10 cada día.
  Aún sin calibrar, pero pipeline funcional.
```

### Mes 3: Dashboard y journal

```
SEMANA 9-10:
  ☐ Dashboard Streamlit MVP:
    - Régimen actual
    - Scores por divisa
    - Señales por par
    - ATR, RSI, precio
    - Sizing sugerido (RPT 0.5% fijo, sin Kelly aún)
  ☐ Stop ATR-based calculado automáticamente

SEMANA 11-12:
  ☐ Trade journal automático (template):
    - Pre-trade: campos requeridos
    - Post-trade: incluyendo "¿La tesis se materializó? Sí/No/Parcial"
  ☐ Journal emocional template (manual)
  ☐ Métricas básicas: hit rate, R-multiple, P&L

DELIVERABLE FIN DE MES 3:
  Dashboard funcional + journal estructurado.
  Listo para empezar a tomar decisiones manuales informadas.
```

### Mes 4: Refinamiento

```
SEMANA 13-14:
  ☐ Reglas de filtro implementadas:
    - Convicción mínima
    - Régimen veto
    - Blackouts NIVEL 1
  ☐ Cálculo de costes (spread + slippage estimado)
  ☐ Edge neto vs costes pre-trade

SEMANA 15-16:
  ☐ Versionado: tag v0.1
  ☐ Tests unitarios > 60% coverage
  ☐ Documentación interna del código
  ☐ Backup cloud automatizado

DELIVERABLE FIN DE MES 4:
  MVP estable v0.1.
  Listo para empezar paper trading manual usando las señales.
```

### Decision gate B → C
```
PARA AVANZAR A FASE C:
  ✓ MVP genera señales coherentes diariamente
  ✓ Dashboard utilizable
  ✓ Journal estructurado funcionando
  ✓ Sin bugs críticos pendientes
  ✓ Decisión: ¿la lógica del modelo es la que esperabas?
     Si no: refinar antes de calibrar (calibrar lógica equivocada = perder tiempo)
```
