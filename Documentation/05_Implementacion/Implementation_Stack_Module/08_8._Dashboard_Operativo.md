## 8. Dashboard Operativo

### 8.1 Implementación con Plotly Dash o Streamlit

```text
RECOMENDACIÓN: Streamlit
  Razones:
    - Más simple para un solo usuario (uno mismo)
    - Reload automático al cambiar código
    - Interfaz Python pura, sin HTML/JS
    - Componentes pre-construidos (gráficos, tablas, métricas)

EJECUCIÓN:
  streamlit run scripts/dashboard.py
  → abre localhost:8501 en navegador
  → reload automático al guardar cambios

ALTERNATIVAS:
  Plotly Dash:  más pro, requiere más código
  Grafana:      mejor para métricas operacionales (no análisis)
```

### 8.2 Layout (referencia §9 Self-Awareness)

```text
TABS / SECTIONS del dashboard:

TAB 1: Equity & P&L
  - Equity curve (12m, 3Y, all)
  - Drawdown plot
  - Métricas live (Sharpe, Sortino, Calmar, MDD, hit rate)
  - P&L por mes (heatmap)

TAB 2: Posiciones
  - Posiciones abiertas con MFE/MAE en tiempo real
  - Exposure neta por divisa
  - Stops y targets visuales
  - Stop temporal countdown

TAB 3: Señales
  - Señal_FX actual por par G10
  - Convicción visual (barra)
  - World View state
  - z-scores top 10 más extremos

TAB 4: Atribución
  - P&L por bloque (Endo, Exo, World View, Timing, Risk, Exec)
  - P&L por driver (top 10 contribuyentes)
  - P&L por par
  - P&L por horizonte
  - P&L por régimen

TAB 5: Self-Awareness
  - Triage diagnóstico actual (A/B/C)
  - Métricas health (rolling 12m, 6m, 3m)
  - Trades últimos 30 días con detalles
  - Lecciones agregadas del journal

TAB 6: Calidad de Datos
  - Health check status por indicador
  - Última actualización por serie
  - β drift por driver
  - Alertas activas
```
