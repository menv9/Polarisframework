## 10. Seguridad

### 10.1 Secrets management

```text
NUNCA commitear:
  ✗ API keys (FRED, broker, Bloomberg)
  ✗ Credenciales de broker
  ✗ Tokens de bots de Telegram
  ✗ Passwords de DB
  ✗ Datos de cuenta bancaria

SIEMPRE en .env (gitignored):
  FRED_API_KEY=xxx
  IB_USERNAME=xxx
  IB_PASSWORD=xxx
  TELEGRAM_BOT_TOKEN=xxx

Cargado con python-dotenv:
  from dotenv import load_dotenv
  load_dotenv()
  
  api_key = os.getenv("FRED_API_KEY")
```

### 10.2 Broker API safety

```text
INTERACTIVE BROKERS (ejemplo):
  - Conexión solo desde IPs whitelisteadas
  - Read-only API key separada de trading API key
  - Trading API requiere password adicional
  - Activar 2FA siempre
  - Trading Permissions limitar a productos permitidos (no margin si no aplica)

REGLA DE ORO:
  - Cuenta de paper trading separada para tests
  - Cuenta real solo recibe órdenes validadas por la real
  - NUNCA usar la cuenta real para experimentar
```

### 10.3 Defensas contra bugs catastróficos

```text
HARD LIMITS en código (no configurables sin commit):
  - Pos_size máximo absoluto (ej: 10% capital, no más)
  - Max trades simultáneos (ej: 8 across todas las capas)
  - Stop_distance mínimo (ej: 0.5× ATR, evita stops absurdamente cercanos)
  - Sanity check de orden antes de enviar:
      ¿el precio del fill está cerca del bid/ask actual?
      ¿el size está dentro de límite del capital?
      ¿el stop está en el lado correcto del entry?

KILL SWITCH:
  Comando para parar TODO inmediatamente:
    scripts/kill_switch.py
  Acciones:
    - Detener scheduler
    - Cancelar todas las órdenes pendientes
    - NO cerrar posiciones abiertas (decisión humana)
    - Alerta crítica al operador
```
