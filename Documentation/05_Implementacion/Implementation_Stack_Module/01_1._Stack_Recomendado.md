## 1. Stack Recomendado

### 1.1 Lenguaje y librerías core

```text
LENGUAJE PRINCIPAL: Python 3.11+
  Razón: ecosistema más maduro para finanzas cuantitativas
         libraries gratuitas suficientes para arrancar
         comunidad activa, abundancia de tutoriales

LIBRERÍAS CORE:
  numpy           → álgebra lineal, vectores, arrays
  pandas          → series temporales, dataframes
  scipy           → estadística, optimización
  statsmodels     → regresión OLS, p-values, VIF, diagnósticos
  scikit-learn    → ML utilities (CV, scaling, métricas)
  pykalman        → Kalman Filter (β dinámico Capa 2)
  arch            → modelos volatilidad (GARCH si necesario)

DATOS:
  yfinance        → Yahoo Finance (gratis, equities + FX + futuros)
  fredapi         → FRED API (datos USA + globales gratis)
  pandas-datareader → wrapper a múltiples fuentes
  requests        → ECB SDW, BIS, otros APIs
  beautifulsoup4  → scraping si necesario (último recurso)

VISUALIZACIÓN:
  matplotlib      → gráficos estáticos
  plotly          → dashboards interactivos
  seaborn         → estadística visual

UTILIDADES:
  pyyaml          → config files (data_sources.yaml)
  python-dotenv   → secrets / API keys
  loguru          → logging estructurado (más simple que stdlib)
  tqdm            → progress bars en backtests largos
  pytest          → tests unitarios

PROGRAMACIÓN:
  apscheduler     → cron jobs Python (scheduler de actualización)
  jupyter         → notebooks para análisis exploratorio

OPCIONAL (avanzado):
  ib_insync       → API Interactive Brokers (broker tier-1)
  ccxt            → APIs múltiples brokers crypto (si extiende)
  numba           → JIT compilation para loops críticos
```

### 1.2 Almacenamiento de datos

```text
ARRANQUE (capital pequeño, simple):
  Parquet files locales
  → 1 archivo por serie histórica
  → comprimido, columnar, rápido para series temporales
  → versión fácil con git LFS
  
Estructura:
  data/raw/fred/CPIAUCSL.parquet
  data/raw/yfinance/EURUSD=X.parquet
  data/processed/zscores_10Y.parquet
  data/processed/scores_endo.parquet

ESCALADO (más datos, más historia):
  SQLite local
  → BD relacional embebida, sin servidor
  → soporta millones de rows fácilmente
  → backups simples (un archivo)

ESCALADO+ (multi-usuario, prod):
  PostgreSQL + TimescaleDB extension
  → optimizado para series temporales
  → particionamiento automático por tiempo
  → SQL completo

NO USAR (en este nivel):
  CSVs (lentos, sin tipos)
  Excel (limites de filas, lentos)
  MongoDB / NoSQL (overkill para datos tabulares)
```

### 1.3 Versionado y gestión

```text
VERSION CONTROL:
  git + GitHub (privado)
  
SECRETS MANAGEMENT:
  archivo .env (NUNCA commitear)
  python-dotenv para cargar
  Para producción seria: HashiCorp Vault o AWS Secrets Manager
  
DEPENDENCY MANAGEMENT:
  poetry (recomendado)  → pyproject.toml + lock file
  pip + requirements.txt → más simple, menos robusto
  
ENVIRONMENTS:
  Python venv o conda
  Aislamiento por proyecto
```
