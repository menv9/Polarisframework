## 9. Testing

### 9.1 Tests obligatorios

```text
UNIT TESTS (pytest):
  - test_zscore.py:        z-score se calcula correctamente, manejo de NaN, ventana
  - test_signals.py:       Score_Endo, Score_Exo, combine, signal generation
  - test_sizing.py:        Kelly, RPT, VolTarget, MIN combinación
  - test_stops.py:         ATR stops, temporal stops, R-multiple
  - test_portfolio.py:     exposición por divisa, correlación, VaR
  - test_circuit_breakers.py: niveles de DD, ramp-down, ramp-up

INTEGRATION TESTS:
  - test_daily_pipeline.py: flujo completo de daily_update.py con datos sintéticos
  - test_backtest.py:       backtest sobre histórico mini con resultado conocido
  - test_no_lookahead.py:   verifica que no hay data leak entre IS y OOS

REGRESSION TESTS:
  - test_known_signals.py:  fechas históricas conocidas → señal esperada
                            (ej: 2008-09-15 Lehman → régimen risk-off detectado)
```

### 9.2 Coverage mínimo

```text
Target inicial:    > 70% line coverage
Target maduro:     > 85%

CRÍTICO TESTAR:
  - Funciones que tocan dinero (sizing, P&L, stops)
  - Cálculos de z-score y β (errores aquí destruyen el sistema)
  - Manejo de fechas (huso horario, business days, holidays)
  - Persistencia (no perder datos en escritura)

OPCIONAL:
  - Visualización (rara vez introduce bugs costosos)
  - Logging (poco crítico)
```
