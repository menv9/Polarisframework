## 5. Backtest Framework

### 5.1 Estructura

```text
RESPONSABILIDADES del motor de backtest:
  1. Iterar histórico día a día (point-in-time, sin look-ahead)
  2. Reconstruir señales con datos disponibles en cada t
  3. Aplicar reglas de timing, sizing, stops
  4. Simular ejecución con costes
  5. Trackear P&L, posiciones, drawdowns
  6. Generar métricas y reportes

REGLAS PARA EVITAR LOOK-AHEAD:
  - Usar datos con publication_lag (no datos revisados ex-post)
  - Training set 7Y, OOS 3Y (sin overlap)
  - Walk-forward con embargo de N días
  - Purging: eliminar muestras de train que solapan temporalmente con test
```

### 5.2 Métricas obligatorias por backtest

```text
Por cada estrategia / set de β probado:
  Sharpe IS, Sharpe OOS
  Sortino IS, Sortino OOS
  Calmar
  Max DD (IS y OOS)
  Time underwater %
  Hit rate
  Profit factor
  Avg trade duration
  IC (mensual y trimestral)
  
  Deflated Sharpe Ratio (corrección por multiple testing)
  Probabilistic Sharpe Ratio
  
  Distribución de retornos: skew, kurtosis
  Tail risk: VaR 95%, CVaR 95%
```

### 5.3 Visualización del backtest

```text
Reporte HTML autogenerado con plotly:
  - Equity curve (linear y log)
  - Drawdown plot (underwater)
  - Distribución de retornos (hist + QQ plot)
  - Rolling Sharpe 12m
  - Heatmap: P&L por mes y año
  - Atribución por bloque (stacked area)
  - Atribución por par
  - Lista detallada de trades con métricas
```
