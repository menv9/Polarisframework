## 3. Pipeline de Datos

### 3.1 Fetcher pattern

```python
# Ejemplo conceptual: fred_fetcher.py
from fredapi import Fred
import pandas as pd
from pathlib import Path

class FredFetcher:
    def __init__(self, api_key: str, cache_dir: Path):
        self.fred = Fred(api_key=api_key)
        self.cache_dir = cache_dir
    
    def fetch(self, series_id: str, start_date: str = "2010-01-01") -> pd.Series:
        """Descarga serie FRED y cachea localmente."""
        cache_path = self.cache_dir / f"{series_id}.parquet"
        
        if cache_path.exists():
            cached = pd.read_parquet(cache_path)
            # Solo actualizar últimos N días
            last_date = cached.index.max()
            new_data = self.fred.get_series(series_id, observation_start=last_date)
            updated = pd.concat([cached, new_data]).drop_duplicates()
        else:
            updated = self.fred.get_series(series_id, observation_start=start_date)
        
        updated.to_frame(series_id).to_parquet(cache_path)
        return updated
```

### 3.3 Health checks (§16.4 endo)

```text
Antes de generar señal cada día, verificar:
  - Cobertura: ≥ 80% datos no nulos en ventana 10Y
  - Frescura: última publicación < 90 días
  - Saltos: ningún jump > 5σ vs std histórica
  - Rangos: valores dentro de min/max histórico × 2
  - Estacionariedad: ADF p < 0.10 tras transformación

Si > 30% de indicadores fallan → no emitir señal ese día.
Loggear todos los fallos para revisión.
```
